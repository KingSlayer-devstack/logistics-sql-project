📦 Warehouse_stock: Logistics Intelligence Project

"The Way Must Be Tried" — Transforming Logistics through Data

📖 The Story Behind the Code
This project is not just a collection of SQL queries; it is the culmination of 5 years on the warehouse floor. My journey began at Portobello, where I first encountered the "phantom locations" and "lost items" that cause massive financial drains.

After moving to industry leaders like Natura & Co and Mercado Livre, I operated the SSI SCHAEFER automation system and realized that the bridge between manual labor and operational excellence is Data Analytics. This repository is my first step in building that bridge.

🚀 Project Objective

The Warehouse_stock project simulates a real-world inventory management system designed to solve the common inefficiencies I witnessed in high-volume logistics corridors:

Eliminating "Phantom Locations" through strict relational integrity.

Optimizing Receiving & Dispatch tracking.

Inventory Accuracy monitoring to prevent "huge losses" in stockrooms.

🛠️ Tech Stack

Language: SQL (Structured Query Language)

Focus: Relational Database Design, Data Integrity, Inventory Logic.

Inspiration: Real-world WMS (Warehouse Management Systems) and automation logic (SSI SCHAEFER/Sequoia).

📈 Professional Experience & Context

My technical approach is grounded in a "ground-up" understanding of the supply chain:

Natura & Co: Experienced in high-level automation and training teams on flow-monitoring.

Mercado Livre: Deep understanding of the "receivement" sector and high-velocity throughput.

Portobello: Foundation in stockroom auditing and inventory reconciliation.

🧩 Key SQL Features (Current & Planned)

[x] Core Schema: Designing tables for Products, Locations, and Movements.

[ ] Inventory Audit Queries: Identifying discrepancies between "system stock" and "physical stock."

[ ] Automation Logic: Triggers to update stock levels in real-time.

[ ] Reporting: SQL views to highlight "stagnant items" or "low-stock alerts."
